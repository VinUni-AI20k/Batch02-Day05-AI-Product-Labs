Mục 2

Dùng thử: Promise vs Reality

🎯

Product hứa gì?

GrabFood hứa giúp tìm và đặt đồ ăn nhanh chóng. Giao diện có danh mục (Cơm, Bún/Phở, Đồ uống...) và thanh search.

👥

User nào được hứa?

Bất kỳ ai muốn đặt đồ ăn giao tận nơi — đặc biệt dân văn phòng bận rộn trong giờ nghỉ trưa.

🧠

Kỳ vọng AI làm gì?

Khi user mô tả mơ hồ: "đồ nước, thanh đạm, gần Quận 1" → kỳ vọng app hiểu ngữ cảnh và gợi ý phù hợp.

💥

Điểm gãy ở đâu?

Lớp Search/Filter — không có Semantic Understanding. App chỉ keyword matching thuần túy.

Evidence thu thập từ thực tế kiểm thử

Test 1 — Failure

Input: "đồ nước thanh đạm"

Kết quả GrabFood: 0 quán. App tìm chính xác chuỗi ký tự "đồ nước thanh đạm" trong tên quán/món → không match → trả rỗng.

Test 2 — Sai lệch

Input: "ăn gì buổi trưa nhẹ nhàng"

Kết quả GrabFood: Hiển thị quán có chữ "nhẹ" trong tên (VD: "Trà sữa nhẹ tênh") — hoàn toàn lệch khỏi intent.

Test 3 — Đối chiếu ChatGPT

Input: "Trưa nay ăn gì đồ nước, thanh đạm ở Quận 1"

Kết quả ChatGPT: Hiểu đúng intent → phân tích loại món (đồ nước), taste (thanh đạm), location (Quận 1) → gợi ý phở, bún, súp. Nhưng quán ảo, sai vị trí.

Test 4 — Happy path

Input: "phở"

Kết quả GrabFood: Nhiều kết quả chính xác. Chỉ hoạt động khi user đã biết muốn ăn gì.

Mục 4

Finding thành Quyết Định Sản Phẩm

Khi user [nhập mô tả mơ hồ: "đồ nước thanh đạm", "ăn gì buổi trưa nhẹ nhàng"],

AI/product [chỉ keyword matching thuần túy, không hiểu ngữ nghĩa/ý định],

hậu quả là [trả 0 kết quả hoặc sai lệch, ép user tự chuyển "cảm giác thèm" → "tên món cụ thể" — gây decision fatigue 20–30 phút],

Lỗi thuộc layer [Intent Understanding + Search/Filter — thiếu Semantic Search],

Nên sửa bằng [bổ sung AI layer giữa user input và search engine].

📋 Product Decision — Đổi gì trong SPEC

  1. Parse mô tả tự nhiên → extract tags (loại món, taste, location, mood)
  2. Map tags vào dataset có gán metadata chi tiết → rank top 3
  3. Hiển thị kèm lý do ngắn gọn vì sao phù hợp
  4. Áp dụng hard filter: dị ứng (bắt buộc) + khoảng cách < 3km
  5. Nếu không đủ data → hỏi 1 câu clarification thay vì trang rỗng
Mục 5

Sơ đồ Luồng: As-is → To-be

💥 As-is — GrabFood hiện tại

👤 User: "tôi muốn ăn đồ nước, thanh đạm, loanh quanh Quận 1"
📱 GrabFood Search: Keyword matching → tìm "đồ nước thanh đạm" trong tên quán
🛑 ĐIỂM GÃY: 0 kết quả. Không suggestion, không hỏi lại, không giải thích.
😤 User: Phải tự nghĩ tên món → gõ "phở" → mất 10–15 phút
✖ Kết quả: Decision fatigue, trải nghiệm kém, user có thể bỏ app.

💡 To-be — AI Food Recommendation

👤 User: "tôi muốn ăn đồ nước, thanh đạm, loanh quanh Quận 1"
🤖 AI Parser: Extract → loại_món: "đồ nước", taste: "thanh đạm", location: "Quận 1"
🔒 Hard Filter: Check dị ứng → loại bỏ. Check khoảng cách < 3km.
📊 Filter dataset 10–20 món → rank theo relevance → chọn top 3
📱 Hiển thị 3 Card kết quả kèm lý do. User chọn trong < 1 phút. ✅
